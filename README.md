# Data Property and Compliance Skill
### 数据要素全生命周期合规审查框架

> ⛔ **发布暂缓 / PUBLISH ON HOLD** —— 本框架原始素材含有**有保密义务的内部培训题库**。在取得书面授权或完成"净室重写"前，**请勿推送到任何公开仓库**。理由与处理路径见 [`PUBLISH-HOLD.md`](./PUBLISH-HOLD.md)。

> 一套覆盖**数据要素全生命周期**的系统化合规自查 / 培训框架：从会计入表到金融证券化，从数据产权登记到可信流通，从场内交易到生态合作。
> 以 AI Agent **Skill** 形式提供（[`SKILL.md`](./SKILL.md)），也可直接当作人工审查 checklist 使用。

## 覆盖范围（五大领域）

| 领域 | 内容 |
|---|---|
| 一 · 数据资产入表 | 会计处理、成本核算、初始/后续计量、减值、估值方法 |
| 二 · 数据资产证券化（ABS） | SPV、资产池、信用增级、现金流、估值难点 |
| 三 · 数据产权登记（核心） | 六维度审查、矛盾识别、法律法规条文级对照 |
| 四 · 可信数据空间 | 隐私计算、区块链存证、访问控制、"数据可用不可见" |
| 五 · 场内交易 | 线上/线下/备案规则、服务流程、登记材料、生态合作 |

外加**跨领域通用审查工具**：PDF 隐藏字段发现、表格间矛盾识别、时间线审计、SaaS 角色辨析、等保 vs CCRC/ISO 辨析、去标识化 vs 匿名化辨析。

## 怎么用

**作为 AI Skill（推荐）**
把 [`SKILL.md`](./SKILL.md) 放进支持 Skills 的 Agent 环境（如 Claude Code / Codex 的 skills 目录），在审查数据要素合规问题时自动触发。它会引导按"六维度 + 法规对照 + 矛盾识别 + 风险矩阵 + 补正清单"的结构产出审查记录。

**作为人工 checklist**
直接按 `SKILL.md` 中的检查项（`- [ ]`）逐项核对申报 / 交易材料，重点关注：来源口径是否自洽、个人信息是否如实申报、权益配置是否一致、安全能力是否到位。

## 关于本开源版本（透明说明）

本仓库是从一份内部实务沉淀中**清理后**对外开源的版本。为可安全公开，已做如下处理：

- ✂️ **剥离来源**：移除"基于某交易所培训题库及实际登记案例提炼"等表述，改为"整理自公开法律法规、国家标准与通用合规方法论"；不包含任何**非公开内部培训资料原文**。
- 🕵️ **去案例化**：删除真实申请人、合同编号、数据产品等可识别信息，所有示例均为**脱敏 / 示意**。
- ⚖️ **法条更正**：修正了若干条号（例如：测绘资质管理应为《测绘法》第二十七条等，而非旧稿误写的"第22条"——第二十二条为不动产测绘管理）；对易变条号统一加注"以现行有效文本为准"。
- 🌐 **去地域绑定**：场内交易规则、生态合作伙伴等改为"以各交易所公开规则为准"的中性表述。

> ⛔ **重要**：经确认，原始素材中的**内部培训题库存在保密义务**。删除"出处声明"与"逐字原文"**并不等于**解除保密——保密保护的是**内容实质**。因此本版本**在取得书面授权或完成净室重写前不得公开发布**。详见 [`PUBLISH-HOLD.md`](./PUBLISH-HOLD.md) 与 [`CITATIONS-REVIEW.md`](./CITATIONS-REVIEW.md)。
